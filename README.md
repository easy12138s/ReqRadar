# 需求透视（ReqRadar）

一个需求从提出到评审再到开发，参与的人各有各的视角，信息经常对不齐。

这个项目想做的是一个中间服务：把需求文档、历史记录、代码库这些地方的信息串起来，在关键节点上给产品、开发、管理者提供一份共用的上下文参考。

**产品经理**提需求时，能先看到技术侧的可能影响范围。  
**部门领导**做评审时，能看到类似历史需求的结局和风险提示。  
**开发者**接手时，能知道这部分代码谁最熟、讨论中提过哪些隐性约束。

不做代码生成，也不做监控。只希望在大家动手之前，信息能更好的对齐。

## 准备怎么做

1. 把需求文档、代码仓库、提交记录这些信息提前扫一遍，做成可检索的索引。
2. 有新需求进来时，自动生成一份简短的参考报告，把相关联的信息列出来。
3. 聊天记录的提取放在本地跑，只输出摘要要点，原文不外传。
