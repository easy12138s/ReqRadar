# 需求透视（ReqRadar）

从需求提出到开发真正理解，中间漏掉的信息太多了。

这个项目想做的事很简单：在动手写代码之前，把散落在各处的重要信息拉出来看一眼。

比如：
- 这需求是不是之前就提过、做过、或者做过又放弃了？
- 改这点东西，实际会碰到哪些代码文件？
- 这部分代码平时是谁在改？谁最熟？
- 群里讨论的时候，有没有提过文档里没写的坑？

不做代码生成，也不是监控工具。就只是帮团队在动手之前，把信息对齐一下。

## 大概怎么做

分两步：

1. 先把代码库、历史需求、提交记录扫一遍，建个索引。
2. 每次来新需求，拿文档跑一遍，吐一份简短报告，把上面那几个问题的答案列出来。

聊天记录里的信息会在本地处理，只提取关键点，原文不会传出去。
