# 需求文档：用户登录系统

## 1. 功能概述

设计并实现一个统一的用户认证系统，支持以下登录方式：
- **邮箱 + 密码登录**：用户使用注册邮箱和密码进行身份验证，支持"记住我"功能
- **第三方 OAuth 登录**：接入 Google 和 GitHub OAuth 2.0 协议，实现一键授权登录
- **双因素认证（2FA）**：在密码验证后，要求用户输入 TOTP 动态验证码（支持 Google Authenticator）

## 2. 安全要求

- **密码强度校验**：密码长度 ≥ 8 位，必须包含大小写字母、数字和特殊字符中的至少三类
- **登录失败锁定**：连续 5 次登录失败后，账户自动锁定 30 分钟，同时发送安全告警邮件
- **会话超时**：普通会话 30 分钟无操作自动过期，"记住我"会话有效期 7 天，会话 Token 使用 JWT + Refresh Token 机制

## 3. 非功能需求

- 登录接口平均响应时间 < 2 秒（P99 < 3 秒）
- 系统需支持 1000 并发登录请求，峰值 2000
- 所有认证日志需保留 180 天，支持审计追溯
- OAuth Token 存储需加密，密钥定期轮换
