## 评审标准

### 质量维度（4 项，V2 必须 ≥ V1）

| 维度 | 评估方法 | 通过标准 |
|------|---------|---------|
| 风险识别准确率 | 标注风险点，比对召回率和精确率 | V2 ≥ V1 |
| 证据溯源完整度 | 检查结论是否有 L0/L1 证据支撑 | V2 ≥ V1 |
| 分析覆盖度 | 标注应覆盖模块，比对命中率 | V2 ≥ V1 |
| 整体质量 | LLM-as-judge 打分 (1-10) | V2 ≥ V1 |

### 成本维度（3 项，全部达标）

| 指标 | 通过标准 |
|------|---------|
| Token 成本比 | V2/V1 ≤ 1.5 |
| Assembly 延迟 | ≤ 2s |
| 总延迟比 | V2/V1 ≤ 1.3 |

### Fail-fast 规则（任一触发直接判负）

| 条件 | 说明 |
|------|------|
| 关键风险完全未识别 | 人工标注的高风险点在 V2 报告中无任何提及 |
| Assembly 延迟 > 5s | 用户不可接受的等待 |
| Token 成本 > V1×2 | 成本失控 |
