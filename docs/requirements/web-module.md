# Web 模块：支持网页操作与在线分析

## 背景

ReqRadar 当前仅支持 CLI 命令行操作，分析结果以 Markdown 文件输出。在实际团队协作中，产品经理、评审者和开发者需要共享分析结果，但 CLI 的使用门槛限制了非技术角色的参与。此外，索引构建和分析过程缺乏可视化进度反馈，用户无法实时了解任务执行状态。

需要为 ReqRadar 添加 Web 模块，提供浏览器端的操作界面，使团队成员能够通过网页提交需求分析、查看分析进度、浏览历史报告、管理项目记忆，从而降低使用门槛、提升协作效率。

## 功能需求

### 1. 项目列表与索引管理

- 首页展示已注册的项目列表，每个项目卡片显示项目名称、描述、最近索引时间和索引状态
- 支持在 Web 端触发 `reqradar index` 命令，选择代码仓库路径和需求文档目录
- 索引构建过程显示实时进度：代码解析进度、文档向量化进度、项目画像构建状态
- 索引完成后展示项目画像摘要（架构风格、技术栈、模块列表）

### 2. 需求分析任务管理

- 支持在 Web 端上传需求文档（Markdown/PDF/DOCX）或直接粘贴需求文本
- 提交分析任务后，通过 WebSocket 实时推送 6 步流程的执行进度（读取→提取→映射→检索→分析→生成）
- 每个步骤显示状态（等待/执行中/完成/失败）和耗时
- 分析任务列表支持筛选（按项目、时间、风险等级）和排序
- 失败的任务支持查看错误日志和重新执行

### 3. 分析报告在线查看

- 分析完成后自动跳转到报告详情页
- 报告页面按章节渲染 Markdown 内容，支持目录导航和锚点跳转
- 影响模块列表中的路径支持点击跳转到代码浏览页面
- 风险等级使用颜色标签可视化（红/橙/黄/绿）
- 支持报告导出为 PDF 和原始 Markdown 文件
- 报告历史版本对比：同一需求多次分析时，高亮显示差异

### 4. 项目记忆可视化

- 术语管理页面：展示术语表（术语、定义、领域、来源），支持搜索、新增、编辑、删除
- 模块管理页面：展示模块列表（名称、职责、核心类、代码摘要），支持编辑模块描述和代码摘要
- 团队信息页面：展示贡献者列表（姓名、角色、负责模块），信息来源标注（Git 分析/手动添加）
- 分析历史时间线：按时间线展示历史分析记录，每条记录关联需求摘要、风险等级、影响模块
- 记忆变更审计：记录每次记忆修改的操作人、时间、变更内容

### 5. 多人协作与权限控制

- 用户注册与登录：支持邮箱密码注册，以及 OAuth2.0 第三方登录（GitHub/Google）
- 角色权限模型：管理员（全权限）、编辑者（可提交分析、编辑记忆）、查看者（只读）
- 项目级别权限：不同用户可被授权访问不同项目
- 操作审计日志：记录所有关键操作（索引构建、分析提交、记忆修改、权限变更）

### 6. API 接口层

- 提供完整的 RESTful API，覆盖所有 Web 界面功能
- API 认证：基于 JWT Token，支持 Token 刷新机制
- API 文档：自动生成 OpenAPI 3.0 规范文档，提供 Swagger UI 交互式调试界面
- Webhook 支持：分析任务完成后可回调指定 URL，便于与 CI/CD 或企业 IM 集成
- 批量操作 API：支持批量提交分析任务、批量导出报告

## 技术约束

- 后端使用 FastAPI + Uvicorn，WebSocket 使用 FastAPI 原生支持
- 前端使用 React + TypeScript，UI 组件库使用 Ant Design
- Web 模块作为独立 Python 包 (`reqradar.web`) 集成到现有项目结构中，不修改核心分析流程
- 数据存储使用 SQLite（轻量部署）或 PostgreSQL（生产环境），通过 SQLAlchemy ORM 抽象
- 认证模块复用现有 LLM 客户端的 OpenAI 兼容接口风格，保持 API 设计一致性
- 与 CLI 完全解耦：Web 模块通过调用 Scheduler 和 Agent 层 API 实现功能，不直接调用 CLI 命令
- 必须向后兼容：不影响现有 CLI 和 MCP Server 的使用方式
- 首次部署时间不超过 10 分钟（Docker Compose 一键启动）

## 非功能性需求

- 性能：报告页面首屏加载时间 < 2s；WebSocket 消息延迟 < 500ms
- 安全：所有 API 接口必须鉴权；敏感操作（删除项目、修改权限）需要二次确认
- 可用性：支持暗色主题；响应式布局适配移动端
- 可部署性：提供 Docker 镜像和 docker-compose.yml；支持环境变量配置
- 可观测性：集成 structlog 结构化日志；提供 /health 和 /metrics 端点
