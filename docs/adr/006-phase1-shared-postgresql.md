# ADR-006: Phase 1 共享 PostgreSQL

## Context

V2 架构设计 7 个微服务，每个服务可能需要独立数据库以实现完全解耦。但初期开发和测试阶段维护 7 个独立 PG 实例成本过高。

## Decision

**Phase 1-8** 所有服务共享一个 PostgreSQL 实例（不同 Schema 或表前缀隔离），**Phase 10** 根据性能数据决策是否拆分。

## Consequence

- **正面**：运维简单（单实例备份/恢复）、跨服务 JOIN 查询可能、Alembic 迁移单一管理
- **负面**：单点故障、无法按服务独立扩缩容、可能存在跨服务查询耦合

## Tradeoff

| 方案 | 放弃原因 |
|------|---------|
| 每服务独立 PG | 运维成本高、本地开发需 7 个容器、初期无性能瓶颈无需拆分、join 查询不便 |
