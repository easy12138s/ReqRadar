新旧 Layout 结构对比

❌ 旧架构 — 双层嵌套 Layout

AppLayout (Sider + Header + Content) ├── Sider (80-200px, 可折叠) ├── Header (Dropdown Avatar) └── Content (margin:24, padding:24) └── SettingsLayout (又一个Layout!) ├── Sider (200px) └── Content └── 各 Setting 页 问题： • 双层 Sider → 水平空间浪费 • 双层 padding → 视觉拥挤 • logout 用 window.location.href 全刷新 • 选中高亮仅精确路径匹配

✅ 新架构 — 水平 Top Bar + 单层内容区

AppShell (统一顶层) ├── TopBar │ ├── Logo "ReqRadar" │ ├── NavItems [Dashboard|项目|分析|设置] │ └── UserMenu (Avatar + Dropdown) ├── Main (单层, padding:24, max-width:1280) │ └── 各页面 └── ErrorBoundary (包裹 Outlet) 特性： • 单层，无嵌套 Layout • 水平导航，释放垂直空间 • Dashboard 作为独立首页 • useNavigate() 替代 location.href • 路径前缀匹配高亮
